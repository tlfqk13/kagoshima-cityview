# UX 개선 기능 기획서

> 작성일: 2026-06-07  
> 목적: 현장에서 스마트폰을 들고 버스를 기다리는 여행자의 경험을 극상으로 올리는 기능 전수 조사

---

## 핵심 가정

이 서비스의 사용자는 **낯선 도시에서 스마트폰 하나로 버스를 타야 하는 외국인 여행자**다.  
그들의 상황:
- 뙤약볕 아래 서 있거나 걸으면서 폰을 본다
- 데이터가 느리거나 없을 수 있다
- 일본어를 모른다
- 구글맵에 의존하는데 위치가 틀려서 왔다
- 시간이 없다. 버스를 놓치면 30분을 기다려야 한다.

이 맥락에서 "사소하지만 임팩트 있는" 기능을 정의한다.

---

## 기능 목록 — 우선순위별

### P0 — 즉시 구현 (이번 스프린트)

#### 1. 지도 앱 연동 링크
**배경:** 구글맵 좌표가 틀려서 이 서비스를 찾아온 사람들이다. 그러나 그들은 여전히 구글맵을 쓴다. 정확한 좌표를 가지고 구글맵/애플맵/네이버맵을 열어주면 **두 세계를 잇는 다리**가 된다.

| 버튼 | 동작 | 대상 |
|------|------|------|
| Google Maps | `https://maps.google.com/?q={lat},{lng}` 새 탭 | 모든 플랫폼 |
| Apple Maps | `https://maps.apple.com/?ll={lat},{lng}&q={stopName}` | iOS Safari |
| Naver Maps | `https://map.naver.com/index.nhn?lat={lat}&lng={lng}` | 한국어 사용자 |

**UI 위치:** StopDetail 하단 — "이 좌표로 열기" 섹션  
**구현 복잡도:** 낮음 (외부 URL 조합)  
**임팩트:** 매우 높음 — 서비스의 존재 이유와 직결

```
┌─────────────────────────────────┐
│  이 좌표로 길찾기               │
│  ┌──────────┐ ┌────────────┐   │
│  │ Google 🗺│ │ Apple Maps│   │
│  └──────────┘ └────────────┘   │
│  ┌──────────┐                   │
│  │ Naver 🗺 │                   │
│  └──────────┘                   │
└─────────────────────────────────┘
```

---

#### 2. 좌표 복사 버튼
**배경:** 지도 앱 링크가 없는 상황(카카오맵, 바이두 등)이나 친구에게 "여기야" 하고 좌표를 보내고 싶을 때.

**UI:** 정류장 GPS 배지 옆에 작은 복사 아이콘  
**동작:** `navigator.clipboard.writeText("{lat}, {lng}")` → 토스트 메시지 "복사됨"  
**구현 복잡도:** 낮음  
**임팩트:** 중간 — 파워 유저에게 유용

---

#### 3. 정류장 URL 공유
**배경:** 일행과 만나는 장소를 공유할 때. "1번 정류장 앞에서 봐" → `/map/stop_01` URL을 카카오톡으로 보내면 상대방이 바로 그 정류장 지도를 연다.

**동작:** Web Share API → 폴백은 URL 클립보드 복사  
```js
navigator.share({
  title: '가고시마 시티뷰 버스 — 텐몬칸 정류장',
  url: `https://domain.com/map/${stop.id}`
})
```
**UI 위치:** StopDetail 헤더 오른쪽 상단 공유 아이콘  
**구현 복잡도:** 낮음  
**임팩트:** 높음 — 바이럴 기전

---

#### 4. 가장 가까운 정류장 자동 하이라이트
**배경:** "지금 내가 어느 정류장과 제일 가까워?"를 앱이 자동으로 알려주면 탐색 필요 없음.

**동작:**  
1. 지오로케이션 버튼 클릭 (이미 있음)  
2. 현재 위치와 20개 정류장의 haversine 거리 계산  
3. 가장 가까운 정류장을 자동 선택 + 지도 flyTo  

**구현 복잡도:** 낮음 (거리 계산 로직 추가만)  
**임팩트:** 매우 높음 — "나 지금 어디야?" 질문을 앱이 해결

---

#### 5. 구글맵 오류 정류장 — 오류 위치도 함께 표시
**배경:** "구글맵이 어디를 잘못 가리키는지" 시각적으로 보여주면 더 이해하기 쉽다. GPS 오류가 있는 정류장(stop_03 등)에 대해 구글맵이 가리키는 틀린 위치도 핀으로 표시.

**UI:** 오류 정류장 선택 시 지도에 반투명 경고 핀을 추가로 표시 + "구글맵은 여기를 가리킴" 레이블  
**데이터:** stops.json에 `googleMapsLat`, `googleMapsLng` 필드 추가 필요  
**구현 복잡도:** 중간  
**임팩트:** 매우 높음 — 서비스의 핵심 가치 시각화

---

### P1 — 다음 스프린트

#### 6. 정류장 검색
**배경:** 20개는 많지 않지만, 목적지 이름으로 검색하고 싶은 사용자가 있다. "사쿠라지마 행 버스 어디서 타?"

**UI:** SidePanel + BottomSheet 상단에 검색 입력창  
**검색 대상:** 정류장 이름 (3개 언어) + 목적지 이름  
**구현 복잡도:** 중간  
**임팩트:** 중간

---

#### 7. 버스 운행 시간표
**배경:** "다음 버스가 언제야?"는 가장 기본적인 질문인데 현재 아무 정보도 없다.

**내용:**
- 첫차/막차 시간
- 배차 간격 (예: 약 30분)
- 시즌별 변경 여부 안내

**데이터 출처:** GTFS의 stop_times.txt — 이미 공개데이터에 포함  
**구현 복잡도:** 중간 (GTFS 파싱 필요)  
**임팩트:** 매우 높음 — 가장 많이 묻는 정보

---

#### 8. 오프라인 모드 안내
**배경:** PWA가 설치되어 있지만, 사용자가 "오프라인에서도 된다"는 걸 모른다.

**UI:**  
- 홈 화면에 "앱 설치 → 오프라인 사용 가능" 배너 (1회 표시)  
- 오프라인 상태일 때 지도 대신 "저장된 정류장 정보 보기" 모드  

**구현 복잡도:** 중간  
**임팩트:** 높음 — 데이터 없는 현장에서 서비스 가치 극대화

---

#### 9. 정류장 사진
**배경:** 지도상 핀이 정확해도 "이게 진짜 거기야?" 확신이 안 서는 경우. 정류장 외관 사진이 있으면 "맞아, 저 파란 간판 건물 앞이네" 확인 가능.

**내용:** 정류장 입구/표지판 사진 1-3장  
**구현 복잡도:** 낮음 (데이터 수집이 더 오래 걸림 — 직접 촬영 필요)  
**임팩트:** 높음

---

#### 10. 도보 경로 미리보기
**배경:** "현재 위치에서 정류장까지 몇 분?" — 현재는 직선거리도 없음.

**동작:**  
1. 현재 위치 → 선택 정류장 직선거리 + 예상 도보시간 표시  
2. Mapbox Directions API로 실제 도보 경로 선  

**구현 복잡도:** 중간 (Mapbox Directions API 추가)  
**임팩트:** 높음

---

### P2 — 백로그

#### 11. 즐겨찾기
자주 이용하는 정류장 저장. localStorage 기반.

#### 12. 다크 모드
밤에 이용하는 여행자용. 시스템 설정 따라가기 + 수동 토글.

#### 13. 지도 팝업 (Tooltip)
정류장 핀 호버 시 이름 팝업. 데스크톱에서 클릭 없이 이름 확인 가능.

#### 14. 노선 버스 애니메이션
버스가 실제로 노선을 따라 이동하는 시각화 (데모/홍보용).

#### 15. QR 코드 생성
정류장별 QR 코드 생성 → 관광과 제안용. 현장 안내판에 붙이는 용도.

#### 16. 접근성 정보
휠체어 접근 가능 여부, 엘리베이터 유무.

#### 17. 혼잡도 안내
피크 시간대 안내 (예: 10~12시 사쿠라지마 방향 혼잡).

#### 18. 카카오맵 연동
카카오맵은 한국 여행자가 많이 씀. `kakaomap://look?p={lat},{lng}` 딥링크.

## 구현 우선순위 요약

```
즉시 (P0) — 1~2일
  ① 지도 앱 연동 링크 (Google/Apple/Naver)
  ② 좌표 복사 버튼
  ③ 정류장 URL 공유
  ④ 가장 가까운 정류장 자동 하이라이트
  ⑤ 구글맵 오류 위치 시각화

다음 스프린트 (P1) — 1주일
  ⑥ 정류장 검색
  ⑦ 버스 운행 시간표
  ⑧ 오프라인 모드 안내
  ⑨ 정류장 사진
  ⑩ 도보 경로 미리보기

백로그 (P2)
  ⑪ 즐겨찾기
  ⑫ 다크 모드
  ⑬ 지도 팝업
  ⑭ 노선 버스 애니메이션
  ⑮ QR 코드 생성
  ⑯ 접근성 정보
  ⑰ 혼잡도 안내
  ⑱ 카카오맵 연동
```

## 파급 효과 vs. 구현 난이도 매트릭스

```
높은 임팩트
│
│   ⑦운행시간표  ①지도앱링크   ④최근접정류장
│                ③URL공유      ⑤오류시각화
│                ⑨정류장사진
│   ⑩도보경로   ②좌표복사
│   ⑥검색
│
│   ⑧오프라인안내
│
│   ⑫다크모드   ⑬지도팝업     ⑪즐겨찾기
│               ⑮QR코드
│   ⑯접근성정보 ⑰혼잡도       ⑭노선애니메이션
│
낮은 임팩트
├──────────────────────────────────────────
낮은 난이도                          높은 난이도
```

**결론:** ①②③④ 는 낮은 난이도 + 높은 임팩트 → P0으로 즉시 구현.  
⑤⑦은 난이도 중간이지만 임팩트가 서비스 가치와 직결 → P0~P1 경계.

## stops.json 데이터 추가 필요 필드

P0~P1 구현을 위해 기존 stops.json에 추가가 필요한 필드:

```json
{
  "id": "stop_03",
  "googleMapsLat": 31.5912,
  "googleMapsLng": 130.5563,
  "photos": [
    "/images/stops/stop_03_front.jpg"
  ],
  "schedule": {
    "firstBus": "09:00",
    "lastBus": "17:30",
    "intervalMinutes": 30,
    "note": "시즌에 따라 변동"
  }
}
```

## 관광과 제안 관점에서의 우선순위

가고시마시 관광과에 이 서비스를 제안할 때 **가장 인상적인 기능 조합**:

1. **⑤ 구글맵 오류 위치 시각화** — "우리가 발견한 문제를 데이터로 증명"
2. **① 지도 앱 연동** — "기존 앱과 호환성, 사용자 친화적"
3. **⑦ 운행 시간표** — "공식 GTFS 데이터를 관광객 친화적으로 재가공"
4. **③ URL 공유** — "카카오톡/라인으로 정류장 공유 → 바이럴"

이 4개 기능이 완성되면 "단순한 지도 앱"이 아닌 "공식 관광 안내 인프라"로 포지셔닝 가능.
