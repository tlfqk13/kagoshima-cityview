# 호텔 제안서 (한국어 참고본)

> `hotels-ja.md`(일본어 정본)의 검토용 미러. 관광과용과 다른 앵글: **"투숙객 서비스로서의 묵료 도구"**.

## 관광과용과의 차이점

| | 관광과용 | 호텔용 |
|---|---|---|
| 프레임 | 공식 채택·공인 안내 | 투숙객 편의 서비스 도입 |
| 상대의 이득 | 시의 관광 진흥·외국인 대응 | 비용 0으로 게스트 서비스 향상, 프런트 부담 감소 |
| 요청 | 공식 소개·QR 게시·데이터 연계 | 프런트에 QR 카드 1장 비치 |
| 핵심 카드 | 정확성 증거(전수 감사) | 호텔별 최근접 정류장 개인화 (No.X까지 계산済) |

## 구조 (ja 정본과 동일)

0. **결론 선행** — 첫 문장에 "QR 카드 무償 설치 제안"을 명시 (일본 제안서 관례: 結論을先に)
1. **제안 취지** — 투숙객이 정류장을 못 찾아 버스를 놓치는 문제 (2026-07 전수 감사 통계 인용) + "한국에서 여행 온 여행자가 개인 개발" 신분 명시
2. **게스트 메리트** — QR 스캔 → 최근접 정류장 정확 위치, 3개 국어, 앱 설치 불요, 오프라인
3. **호텔 메리트** — 비용 0, "정류장 못 찾음" 문의·안내 부담 감소, 한국인 게스트 대응
4. **설치 요청** — 호텔별 최근접 정류장 명시(No.[ ] 개인화 칸) + A6 카드 인쇄 비치 + **인쇄·라미네이팅 완료품 배송 옵션(送料のみ負担)**
5. **万が一の場合の対応** — 무의무·무계약·묵료 + "언제든 연락 없이 카드를 낼 수 있습니다"(탈출구) + 장애 시 24시간 이내 대응 약속 + 별지 トラブル時のご案内 참조
6. **데이터 취급** — CC BY 4.0, 출처 명기, 비영리
6. **연락처 + 맺음말** — fkffksk20@gmail.com + "ご検討賜りますよう何卒よろしくお願い申し上げます"
7. **부록A** — 발송 메일 예문 (件名+본문 템플릿)
8. **부록B** — 주요 호텔 10곳 × 최근접 정류장 매핑표 (직선거리, 2026-07-20 산출)

> 작성 근거: 일본 비즈니스 제안서 가이드라인 실측 검증 (bizroute 提案書の書き方, MoneyForward 提案書構成) — 결론선행·구성 6요소·맺음말·메일 매너 반영 완료 (2026-07-20)

## 사용 시 주의

- 호텔마다 No.[ ] 와 [停留所名], [ホテル別カードURL]을 채워서 개인화 (예: 시로야마 호텔 → No.7 城山, 카드 URL `/card/stop_07`)
- 다이와 로이넷·오리엔탈 익스프레스는 지오코딩 실패로 "현地 확인" 표기 — 발송 전 수동 확인 필요
- 도메인 미런칭 상태이므로, 실제 발송은 사이트 런칭 이후
